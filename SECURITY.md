# Security Policy – CYFRAM Library

[![Language](https://img.shields.io/badge/language-🇬🇧%20English-informational)](./LANGUAGE.md)

**Purpose:** Voluntary security management and reporting (CRA practice)

## 1. Supported Versions
| Version | Track | Supported |
| ------- | ----- | --------- |
| 1.2.0   | Legacy | ✅ (frozen, security fixes only) |
| <1.2.0  | Legacy | ⚠️ |
| 2.0.0   | Pro (CYFRAM_Pro) | ✅ |

## 2. Reporting a Vulnerability
- Contact: `mailto:artkeller@gmx.de`
- Expected response: 72h
- Process: Evaluate → Assign severity → Publish patch → Update users
- Applies equally to both tracks (Legacy `src/` and Pro `pro/src/`); please
  state which track and which file(s) are affected when reporting.

## 3. Patch Management (PoC)
- Maintain list of affected versions
- Track applied patches
- Optional: provide signed releases / checksums

---

> Following this policy ensures that security issues are handled responsibly, maintaining trust for users and contributors.
